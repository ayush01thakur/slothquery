Analytical Integrity.
Zero database leaks.

Everything runs locally on your workstation. No external API feeds can view your private schema structures.

01

Context Pruning

Automatically maps and restricts schema feeds to only include definitions related to the user's specific request. Keep prompts tidy.

02

Dialect Validation

Syntactic verification checks run in the background using SQLGlot, ensuring generated logic fits Trino, Snowflake, or PostgreSQL rules.

03

Instant Re-indexing

Editing query context or playbooks triggers real-time ChromaDB updates. Human revisions take top-rank retrieval priority instantly.

04

Portable User Data

Application logic separates cleanly from user data. Database and vectors reside inside your home directory. Updates won't affect files.

Getting Started

Set up SlothQuery locally in four quick steps.

1

Install Package

Open your standard command line interface (Terminal or CMD) and execute: pip install slothquery

2

Launch & Configure API

Start the application: slothquery This opens your default browser. Go to settings / configurations to set your LLM API provider, test the connection, and save it.

3

Build or Import Knowledge

Build your analytical knowledge base inside the Knowledge Studio, or go to Settings & Configurations to import a `.slothkb` file shared by someone in your organization.

4

Chat & Understand Logics

Start messaging the agent to generate custom SQL queries, understand business metrics formulas, or inspect table relationships with context-aware accuracy.
